# Git Tags And Releases

## What This Chapter Is About

This chapter teaches how tags mark important points in history and how releases help package versions on GitHub.
The goal is to help you use Git and GitHub with more confidence when the workflow gets a little more real.

## Real-World Analogy

A Git tag is like putting a sticky tab on an important page in a notebook. A release is like wrapping that page in a labeled package so others know it is a version they can use.

## Key Ideas

- Tags point to important commits such as version milestones.
- Annotated tags are more descriptive than lightweight tags.
- GitHub releases often build on tags by adding notes and downloadable assets.

## Why This Matters

Git gets much easier when you stop treating commands like magic words.
You do not need every command at once, but you do need to know which problem each one is meant to solve.
That is what this chapter is helping you practice.

## Common Mistakes

- creating tags with unclear names
- forgetting to push tags after creating them locally
- treating releases like an afterthought with no notes or context

## Practice Tip

Run commands in a throwaway repository first when the action feels risky.
That gives you room to learn without worrying about breaking something important.
