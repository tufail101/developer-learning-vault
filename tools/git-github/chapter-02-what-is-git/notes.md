# Chapter 2: What Is Git

## What Git Is

Git is a version control system.

It helps developers track code changes on their computer.

Git works locally first, which means you can use it before putting anything on GitHub.

## Real-World Analogy

If version control is the idea of tracking history, Git is the actual tool that keeps the history notebook.

## What Git Tracks

Git can track:

- added files
- changed files
- deleted files
- snapshots of your project over time

## Why This Matters

Git is one of the most important tools in professional software development.

You will use it for:

- personal projects
- team projects
- bug fixes
- pull requests

## Example Terms

- repository: a Git-tracked project folder
- commit: a saved project snapshot
- branch: a separate line of work

## Best Practices

- learn Git locally before worrying about GitHub
- make frequent meaningful commits
- understand status before using more advanced commands

## Common Mistakes

- confusing Git with GitHub
- memorizing commands without understanding the workflow
- making commits only after huge changes
