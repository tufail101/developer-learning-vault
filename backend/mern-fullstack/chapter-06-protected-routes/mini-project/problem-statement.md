# Mini Project — Protected Profile Route

## What To Build
Create a protected profile route in the backend and a matching frontend request that handles both success and denial.

## What It Should Do
- Block unauthenticated access
- Return a useful success response for authenticated access
- Handle unauthorized cases clearly on the frontend

## What You Must Use
- auth-check middleware
- one protected backend route
- frontend behavior when access is denied

## Stretch Goal
Show a `Please log in` message in the frontend when access is denied.
