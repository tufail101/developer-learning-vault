# Chapter 8: Full Git Workflow Mini Project

## Goal

Bring together everything you learned:

- initialize a repo
- create files
- commit changes
- create a branch
- merge a branch
- push to GitHub

## Why This Matters

Git becomes much easier once you stop seeing commands as separate facts and start seeing them as one connected workflow.

## Real-World Analogy

This is like practicing the full route after learning individual driving skills.

## Recommended Flow

1. create project folder
2. `git init`
3. add files
4. commit
5. create feature branch
6. make another commit
7. merge into `main`
8. push to GitHub

## Best Practices

- keep each step clear
- check `git status` often
- commit before switching context
- push after the local workflow is stable

## Common Mistakes

- skipping status checks
- merging unfinished work
- pushing before understanding the local history
