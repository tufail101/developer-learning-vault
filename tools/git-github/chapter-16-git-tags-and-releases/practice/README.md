# Practice — Git Tags And Releases

## Exercise 1
Create a fake version tag like `v0.1.0` in a throwaway repo and inspect it with `git tag`.

## Exercise 2
Write a short release note for a pretend project update that includes one feature and one bug fix.

## Exercise 3
Compare a lightweight tag and an annotated tag in your notes so you remember why they differ.
