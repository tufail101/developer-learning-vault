# Instructions

1. Read the chapter notes again and list the main backend ideas you need before writing code.
2. Build the smallest working version first so you can test the core behavior quickly.
3. Add one feature at a time and test after each step.
4. Keep your file names and route names clear enough that someone else could follow them.
5. If something breaks, check the terminal output and the exact request or response before changing random lines.
6. When you finish, explain the full request flow to yourself from start to response.
