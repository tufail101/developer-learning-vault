# Git Stash

## What This Chapter Is About

This chapter teaches how to temporarily save unfinished work with `git stash` when you need to switch context.
The goal is to help you use Git and GitHub with more confidence when the workflow gets a little more real.

## Real-World Analogy

Stashing is like putting your half-finished craft project in a labeled box so you can clear the table for something urgent and come back later.

## Key Ideas

- `git stash` saves current uncommitted changes for later.
- `git stash list` helps you see saved stashes.
- `git stash pop` brings changes back and removes the stash entry if it applies successfully.

## Why This Matters

Git gets much easier when you stop treating commands like magic words.
You do not need every command at once, but you do need to know which problem each one is meant to solve.
That is what this chapter is helping you practice.

## Common Mistakes

- stashing work with no message and forgetting what it was
- treating stash like long-term storage instead of a temporary tool
- popping a stash without checking what branch you are on

## Practice Tip

Run commands in a throwaway repository first when the action feels risky.
That gives you room to learn without worrying about breaking something important.
