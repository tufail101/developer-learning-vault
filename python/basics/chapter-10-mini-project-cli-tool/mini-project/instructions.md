# Instructions — Command Line Study Tracker

1. Create a file such as `study_tracker.py`.
2. Write a function that prints the menu options.
3. Create an empty list to store tasks.
4. Use a `while` loop so the menu keeps showing until the user exits.
5. Add one branch for adding a task.
6. Add one branch for viewing tasks.
7. Add one branch for exiting the program.
8. Test each menu option before adding the stretch goal.

## Suggested Build Order

1. Print the menu first.
2. Add the task list second.
3. Add the add-task flow third.
4. Add the view-tasks flow fourth.
5. Add the exit option last.

## What To Check Before You Stop

1. Can the user add more than one task?
2. Does view-tasks show all saved tasks?
3. Does the loop stop cleanly when the user chooses exit?
4. Does the tool still behave nicely when no tasks exist yet?
