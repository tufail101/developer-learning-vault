# Mini Project — Books REST API

## What To Build
Create a complete small REST API for books that another app could consume later.

## What It Should Do
- Support at least four routes: list, single item, create, and delete
- Use JSON for every response
- Parse request bodies with `express.json()`
- Return clear status codes for success and not-found cases

## What You Must Use
- multiple routes
- JSON responses
- route params and request body parsing

## Stretch Goal
Add an update route with `PUT /books/:id` and return the updated book.
