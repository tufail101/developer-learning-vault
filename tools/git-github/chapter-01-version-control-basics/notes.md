# Chapter 1: Version Control Basics

## What Version Control Means

Version control is a way to track changes in your code over time.

It helps you answer questions like:

- what changed?
- when did it change?
- who changed it?
- how do I go back if something breaks?

## Real-World Analogy

Imagine writing a book and saving only one copy.
If you make a mistake, it becomes hard to recover the old version.

Version control is like keeping a history of every meaningful draft.

## Why Developers Use It

Without version control:

- it is easy to lose work
- teamwork becomes confusing
- bugs are harder to trace

With version control:

- changes are recorded
- teamwork is safer
- history is visible

## Example Idea

You build a website and later change the navbar.
With version control, you can compare the old version and the new version instead of guessing what changed.

## Best Practices

- save meaningful change history
- make small logical updates instead of giant random ones
- write clear commit messages

## Common Mistakes

- making many changes without saving a version
- treating version control like a backup only
- writing useless messages like `update` or `stuff`
