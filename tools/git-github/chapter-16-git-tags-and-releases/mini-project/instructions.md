# Instructions — Version Milestone Release

1. Start in a safe practice repository if the commands feel risky.
2. Read the chapter notes once before you begin.
3. Work through the project one step at a time instead of jumping ahead.
4. Check the output after each command so you understand what changed.
5. Write down any step that felt confusing so you can revisit it later.

## Suggested Build Order

1. Set up the repository or branch state you need.
2. Complete the main workflow step for this chapter.
3. Confirm the result with `git status`, GitHub, or both.
4. Finish the remaining required steps.
5. Try the stretch goal only after the main path feels clear.
