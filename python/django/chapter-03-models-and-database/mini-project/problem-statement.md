# Mini Project — Blog Post Data Model

## What To Build
Design the data model for a small blog app in Django.
You are not building pages yet.
You are designing the `Post` model that the rest of the app will depend on.

## What It Should Do
- define a `Post` model with fields for title, body, and creation time
- include at least one field that helps with blog workflow, like `is_published`
- be clear enough that another person could understand what one blog post record looks like

## What You Must Use
- a Django model class
- at least four fields
- sensible field types
- the migration commands you would run after saving the model

## Stretch Goal
Add a `slug` field and write one sentence explaining why blog URLs often use slugs.
