# Mini Project — Change History Demo

## What To Build
Create a tiny repository that shows how Git tracks changes over time.

## What It Should Do
- Start with one simple file and commit it
- Change the file twice and create two more commits
- Use `git log --oneline` to show the full history

## What You Must Use
- multiple commits
- clear commit messages
- history inspection with `git log`

## Stretch Goal
Use `git diff` to compare the latest changes before committing.
