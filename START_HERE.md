# Start Here

Welcome to the repo.

This learning system is built to take you from complete beginner to someone who can build real projects, read existing code, and keep growing without depending on random tutorial hopping.

You do not need to feel ready before you start.
You only need a browser, a text editor, and the willingness to keep showing up.

## What You Need Before You Start

- a web browser like Chrome, Edge, or Firefox
- a text editor like VS Code
- a folder on your machine where you can keep your practice files

Helpful later, but not required on day one:

- GitHub account
- Node.js
- Git

## Your First 7 Days

### Day 1

- Read [MACHINE_SETUP.md](./MACHINE_SETUP.md)
- Read [HOW_TO_USE_THIS_REPO.md](./HOW_TO_USE_THIS_REPO.md)

- Finish HTML chapter 1 if needed

### Day 3

- Build one very small page structure from memory
- Compare your version against the notes and fix mistakes

### Day 4

- Study HTML chapter 4
- Add links and images to yesterday's practice file

### Day 5

- Study HTML chapter 5
- Build one list and one table by hand
- Complete the chapter practice work

### Day 6

- Study HTML chapter 6
- Build a simple form with labels, inputs, and a submit button

### Day 7

- Review the week's notes
- Revisit the hardest chapter once more
- Complete the HTML mini-project or a simplified version of it
- Write a short note about what felt confusing and what feels clearer now

## Weekly Study Schedule Template

Use this if you want structure without overloading yourself:

- Monday: read notes and run examples
- Tuesday: re-type examples and change them
- Wednesday: complete practice exercises
- Thursday: review mistakes and weak spots
- Friday: continue the next chapter
- Saturday: mini-project or recap build
- Sunday: light review or full rest

If you have less time:

- 30 to 45 minutes per day is enough to make progress
- consistency matters more than long study sessions

## How To Use This Repo Without Burning Out

- do not try to "finish" many chapters in one sitting
- do not collect notes you never apply
- build something small after every lesson
- accept that confusion is part of learning, not proof that you are bad at it
- rest before you hate the process

Good rule:
when your brain feels full, stop consuming and start doing something smaller.

## When You Feel Stuck

- go back one chapter and make sure the previous idea is solid
- shrink the task until it feels possible again
- copy the example, then change one line at a time
- use the browser dev tools and read the error message slowly
- explain the bug out loud in plain language before trying random fixes
- take a break and come back with a smaller goal

Being stuck does not mean you are failing.
It usually means your brain is trying to connect several new ideas at once.
