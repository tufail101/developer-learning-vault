# Mini Project — Complete Team Workflow Simulation

## What To Build
Simulate a full team-style workflow: create a repository, make a feature branch, open a pull request, review it, and merge it.

## What It Should Do
- start from a clean repository and push an initial version
- use a feature branch for one focused change
- include a pull request, review step, and final merge

## What You Must Use
- branch creation and push
- pull request and review flow

## Stretch Goal
Add a second tiny feature branch after the first merge so you can compare how the workflow feels the second time.
