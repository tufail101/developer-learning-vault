# Mini Project — Open Source Contribution Flow

## What To Build
Map out a complete fork-based contribution flow for a small open-source change.

## What It Should Do
- show the fork, clone, branch, and PR steps
- explain the role of `origin` and `upstream`
- keep the contribution small and reviewable

## What You Must Use
- fork workflow steps
- upstream sync awareness

## Stretch Goal
Add one note about reading contribution guidelines before making changes.
