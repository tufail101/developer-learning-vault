# Mini Project — Version Milestone Release

## What To Build
Mark a small project milestone with a Git tag and a short release note.

## What It Should Do
- create a readable version tag
- connect the tag to a meaningful project point
- include short release notes for other people to read

## What You Must Use
- one tag
- one release note

## Stretch Goal
Add a second tag later and compare how the project changed between versions.
