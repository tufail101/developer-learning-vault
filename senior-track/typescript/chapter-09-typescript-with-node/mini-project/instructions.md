# Instructions — Typed Config And Data Helpers

1. Start with the smallest useful type or helper instead of trying to model the whole project at once.
2. Write the main data shape first.
3. Add one function or one usage example that depends on that type.
4. Test whether the code still makes sense when you change one property.
5. Add the stretch goal only after the main version feels clear.

## Suggested Build Order

1. Define the main type or response shape.
2. Add one helper function or one realistic usage example.
3. Add a second related type or action.
4. Handle one obvious error or missing-data case in the design.
5. Clean up names so another person could understand the file quickly.
