# Mini Project — Branching Strategy Plan

## What To Build
Create a small written plan for how a team would use branches on a real project.

## What It Should Do
- name the main long-lived branch or branches
- describe how feature work gets merged
- explain how the team keeps branches from drifting too far apart

## What You Must Use
- at least one clear branch strategy
- reasons for the choices you made

## Stretch Goal
Add a short section for how hotfixes would be handled when a bug appears in production.
