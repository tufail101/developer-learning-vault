# Practice — Basic Git Workflow

## Exercise 1
Create a file, run `git status`, and identify which files Git sees as changed.

## Exercise 2
Stage the file, commit it with a clear message, and check `git log --oneline` to confirm the commit exists.

## Exercise 3
Edit the same file again, compare `git status` before and after `git add`, and explain what changed in the workflow.

## Done Checklist
