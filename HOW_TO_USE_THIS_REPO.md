# How To Use This Repo

Use each chapter in this order:

1. Read `notes.md`
2. Open the files inside `examples/`
3. Complete the tasks in `practice/`
4. Check off `todo.md`
5. Finish the `mini-project/`

## Best Learning Loop

- Read for 20 to 30 minutes
- Code for 40 to 60 minutes
- Explain what you learned in your own words
- Come back the next day and rebuild from memory

## What To Do When You Feel Stuck

- Re-read the example slowly
- Type code by hand instead of copy-pasting
- Change one thing at a time
- Use browser dev tools
- Ask: "What is this line doing?"

## A Good Weekly Pattern

- 4 days learning
- 2 days building
- 1 day reviewing and writing notes
