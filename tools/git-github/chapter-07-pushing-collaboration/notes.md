# Chapter 7: Pushing, Pull Requests, And Collaboration

## Pushing Code

After working locally with Git, you can send your code to GitHub.

Common commands:

```bash
git remote add origin https://github.com/username/project.git
git push -u origin main
```

## Pull Requests

A pull request is a request to merge one set of changes into another branch.

Teams use pull requests to:

- review code
- discuss changes
- merge safely

## Real-World Analogy

Pushing is like sending your updated notebook to the shared shelf.
A pull request is like asking a teammate to review your new pages before adding them to the official version.

## Why This Matters

This is how professional collaboration often works.

## Best Practices

- push often enough to avoid losing work
- make pull requests focused and easy to review
- write clear titles and descriptions

## Common Mistakes

- pushing unfinished unrelated work together
- opening huge pull requests
- skipping review habits
