# Chapter 1: Introduction To Node.js

## What Node.js Is

Node.js lets us run JavaScript outside the browser.

Until now, JavaScript in this repository mostly ran in the browser to:

- update the page
- respond to clicks
- fetch data

With Node.js, JavaScript can also run on the backend to:

- read files
- start servers
- handle requests
- return data

## Real-World Analogy

Think of the browser as the front desk where users interact with an app.
Node.js works more like the office behind the desk where requests are processed and records are handled.

## Why This Matters

If you want to become a full-stack web developer, Node.js is one of the most beginner-friendly ways to learn backend development because it lets you keep using JavaScript.

## First Example

```js
console.log("Hello from Node.js");
```

Run it with:

```bash
node app.js
```

## What Changes From Browser JavaScript

In the browser, you get:

- `document`
- `window`
- DOM APIs

In Node.js, you instead get access to:

- file system tools
- server tools
- path tools
- module system

## Best Practices

- start with small scripts
- understand the difference between browser and server code
- keep terminal output readable

## Common Mistakes

- expecting `document` to exist in Node.js
- confusing frontend and backend responsibilities
- trying to learn everything at once
