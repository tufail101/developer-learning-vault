# Portfolio Checklist

A good junior portfolio proves that you can build, finish, explain, and polish projects. It does not need twenty half-finished apps.

## What Your Portfolio Needs

- a short, clear introduction
- 2 to 4 finished projects
- screenshots or previews
- live demo links
- GitHub repo links
- a short explanation of what you built and why
- a short explanation of the stack used
- responsive design
- clean navigation
- contact information

## What Makes A Strong Project

- solves a clear problem
- looks intentional, not tutorial-default
- includes realistic states
- has clean naming and structure
- can be explained confidently

## Project Ideas That Read Well

- dashboard with filtering and charts
- e-commerce UI with cart logic
- blogging or CMS-like interface
- task or habit tracker with persistence
- frontend app that consumes a real API

## Common Mistakes

- too many unfinished projects
- generic tutorial clones with no personal decisions
- no mobile responsiveness
- broken links
- poor spelling or rushed writing
- no explanation of your contribution

## Project Description Template

For each project, include:

- what the project does
- which problems you solved
- the main technologies used
- one or two technical challenges
- one thing you would improve next

## Portfolio Home Page Checklist

- name and role
- short intro paragraph
- project section
- skills section
- contact section
- links to GitHub and LinkedIn or equivalent

## Final Test

Ask yourself:

- would a recruiter understand what I can do in under one minute?
- do my projects look finished?
- can I explain the technical choices without guessing?
