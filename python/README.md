# Python Track

This track teaches Python from the ground up, then shows how Python fits into web work.

You start with the basics.
Then you move into web scripting.
After that, you build backend ideas with Flask and get a first look at Django.

If you are learning on your own, this matters a lot:
you do not need to guess where to begin.
You can follow the track in order.

## Start Here

If you are brand new to Python, begin with:

- [basics](/home/tufail/Dev/Study/web-dev-learning-system/python/basics)

After that, move in this order:

1. [web-scripting](/home/tufail/Dev/Study/web-dev-learning-system/python/web-scripting)
2. [flask](/home/tufail/Dev/Study/web-dev-learning-system/python/flask)
3. [django](/home/tufail/Dev/Study/web-dev-learning-system/python/django)

That order works well because each module builds on the one before it.

## What You Need

You only need a few things to start:

- Python 3
- a text editor like VS Code
- a terminal

You do not need to know another language first.

## Module Map

### [basics](/home/tufail/Dev/Study/web-dev-learning-system/python/basics)

This is where you learn how Python works:

- writing and running `.py` files
- variables and data types
- conditionals and loops
- functions
- lists, dictionaries, and tuples
- file handling
- error handling
- modules and packages
- object-oriented basics
- one small command-line project

If you skip this module, the later web chapters will feel much heavier than they need to.

### [web-scripting](/home/tufail/Dev/Study/web-dev-learning-system/python/web-scripting)

This module shows how Python interacts with the web from the outside.

You will learn:

- what role Python plays in web development
- how to make HTTP requests
- how to work with JSON data
- how HTML scraping works at a basic level
- how to build a small data-fetching script

This module is a nice bridge between Python basics and backend frameworks.

### [flask](/home/tufail/Dev/Study/web-dev-learning-system/python/flask)

This is the first backend framework path.

You will learn:

- routes and views
- request and response flow
- templates
- forms and POST requests
- SQLite basics
- REST API basics
- simple authentication ideas

Flask is a good first backend framework because it lets you see the moving parts clearly.

### [django](/home/tufail/Dev/Study/web-dev-learning-system/python/django)

This module shows a more structured Python web framework.

You will learn:

- what Django is
- project and app structure
- models and database basics
- views and URLs
- templates
- Django REST Framework basics
- one small Django API project

Django can feel heavier at first, but that structure starts to make sense once you already know the basics.

## A Good Way To Study This Track

Try this simple rhythm:

1. read the chapter notes
2. run the example file
3. do the practice exercises yourself
4. build the mini-project without copying line by line

If something feels hard, that does not mean you are failing.
It usually means you are right at the edge of new understanding.

## When To Move On

Move to the next chapter when you can do these two things:

- explain the current idea in plain words
- change the example a little and still understand what happened

You do not need perfect memory before moving forward.
You just need real contact with the idea.

## End Goal

By the end of this track, you should be able to:

- write useful Python scripts
- read and write files
- organize code across multiple files
- build small command-line tools
- make HTTP requests and work with web data
- build beginner-friendly backend apps with Flask
- understand the shape of a Django project

That is a strong base for using Python in real projects.
