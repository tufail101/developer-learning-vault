# Mini Project — Small Feature PR Simulation

## What To Build
Simulate opening a pull request for a small feature and preparing it for review.

## What It Should Do
- use a focused branch with one clear purpose
- include a readable PR title and description
- list how the change was tested before asking for review

## What You Must Use
- a pull request summary
- a short testing section

## Stretch Goal
Write two example review comments and then write respectful replies to them.
