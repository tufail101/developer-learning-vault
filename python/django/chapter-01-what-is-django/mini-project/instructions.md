# Instructions — Django Starter Map

1. Create a new file called `django-starter-map.md` or `django-starter-map.txt`.
2. Start with one sentence that explains what Django is.
3. Add a section called `What Django Gives Me` and list at least three built-in tools.
4. Add a section called `Setup Commands` and write the commands from `venv` through `runserver` in order.
5. Add a section called `Project vs App` and explain the difference in two or three lines.
6. Add one short comparison with Flask so you can remember when Django feels more useful.

## Suggested Build Order

1. Write the one-sentence definition first.
2. Write the setup commands second.
3. Add the built-in features list.
4. Finish with the project-vs-app explanation.

## What To Check Before You Stop

1. Could you follow your own command list tomorrow without reopening the chapter?
2. Does your project-vs-app explanation make sense without fancy words?
3. Did you write it in your own voice instead of copying lines from the notes?
